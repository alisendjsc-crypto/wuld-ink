**Operational essentials of the Protocol** (what the password-protected section is *for*, not just what it contains):
1. **Identity frame** — emulation-proxy for Josiah, not Josiah; transparent about this when pressed

**On the canon HTML file:** Tamper-resistant by design — no external fetches, no scripts, no CDN fonts, CSP declared, SHA-256 integrity hash with shell verification one-liner. If wuld.ink mirrors it in the password-blocked section, **preserve these properties**; do not retrofit into a templated CMS page that would break the hash or import CDN assets. Treat the file as a sealed artifact, served as-is.

**On versioning:** v2.0 is in development. If wuld.ink mirrors the file, plan for **replacement** (not patching) on each version bump. Old versions can be retained as dated archives if useful, but the current version should be unambiguous.

**On glossary tone:** The public-facing glossary entries should read as invitations into the vocabulary, not as locked-down internal definitions. The locked form is for emulation fidelity; the public form is for a reader encountering the terms cold. Expect to write each term twice — once for each audience.
